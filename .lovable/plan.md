

# Remove the "+" Pattern Once and For All

## Root Cause
The "+" cross pattern visible across the entire page is NOT in the hero section -- it's in `CreatorFooter.tsx` (line 106-112). The footer element at line 105 has no `position: relative` or `overflow: hidden`, but the pattern overlay inside it uses `position: absolute; inset: 0`. This causes the pattern to escape the footer and cover the entire page from the nearest positioned ancestor, which is the page-level container.

## Fix (Simple and Clean)

**File: `src/components/preview/CreatorFooter.tsx`**

1. **Delete the entire pattern overlay block** (lines 106-112) -- the SVG that draws the repeating "+" shapes
2. **Add `relative overflow-hidden`** to the `<footer>` element as a safety measure, so nothing can ever leak out again
3. **Also fix the remaining Dutch text**: "Preview gegenereerd door" on lines 66 and 206 should become "Preview generated by"

That's it. One file, three small changes, pattern gone forever.

## Technical Details

The offending code being removed:
```html
<!-- This draws repeating "+" symbols across the entire viewport -->
<div class="absolute inset-0 opacity-5 pointer-events-none"
  style="background-image: url('data:image/svg+xml,...plus-pattern...')">
</div>
```

The footer tag gets `relative overflow-hidden` added to its className to prevent any future absolute-positioned children from escaping.
